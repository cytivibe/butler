# Contributing to Butler

Contributions are welcome. This document covers the process and legal requirements.

## How to contribute

1. Fork the repository
2. Create a branch for your change
3. Make your changes
4. Run tests: `go test ./...`
5. Submit a pull request

## Code guidelines

- Follow existing code style and patterns
- Add tests for new functionality
- Keep changes focused — one concern per PR

## Contributor License Agreement

All contributions are accepted subject to the [Contributor License Agreement](CLA.md). By submitting a pull request, you indicate that you have read, understood, and agree to the terms of the CLA.

## Questions

Open an issue for bugs, feature requests, or questions.
